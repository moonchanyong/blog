This project specifies interoperability of common algebraic structures

Functor

a functor is a map between categories.

collection의 object들이 다른 타입으로 맵핑 될 수 있음.

Definition

  1. identity function이 제공되면 기존과 같음
  1. F(f(g(x)))가 map을 g, f순으로 한 것과 같음

fantasy-land/map method

fantasy-land/map :: Functor f => f a ~> (a -> b) -> f b

Functor는 반드시 fantasy-land/map 메서드를 제공한다. 하나의 argument만 받는다.

u['fantasy-land/map'](f)
  1. f는 함수이다.
    1. f가 함수가 아닌경우에 fantasy-land/map 의 동작은 명시되어있지 않다.
    1. f는 어느 값이라도 반환한다.
    1. No parts of f's return value should be checked.
  1. fantasy-land/map 은 반드시 같은 functor를 반환 해야한다.

Snippet

class Functor<A> {
  iterator = function*(){...};
   
  map(f: A => B): Functor<B> {
    new Functor()
		for(const iter : iterator) {
      f(iter)
    }
  }
}

Foldable

Definition

  1. ((acc, x) ⇒ acc.concat([x]), [])은 list, 기존이 다른 iterable이라도 reduce 동작은 같음

foldable 은 fantasy-land/reduce 메서드를 가진다. fantasy-land/reduce 는 2개의 arguments를 가진다.

u['fantasy-land/reduce'](f, x)

  1. f 반드시 bifunction이어야 한다.
    1. 만약 f 가 함수가 아니면 fantasy-land/reduce 는 명세되어있지 않다.
    1. f 의 첫 argumnet는 x와 타입이 같아야한다.
    1. f는 반드시 x와 같은 타입을 반환해야한다.
    1. No parts of f's return value should be checked.
  1. x는 초기 누산을 위한 값이다.
    1. No parts of x should be checked.

Filterable

Definition

  1. 분배법칙
  1. 항등원
  1. annihilation operator ⇒ 소멸작용소라고 함

fantasy-land/filter :: Filterable f => f a ~> (a -> Boolean) -> f a

Filterable 은 반드시 fantasy-land/filter 메서드를 가진다.

fantasy-land/filter 는 한개의 전달인자를 받는다.

v['fantasy-land/filter'](p)

  1. `p`는 반드시 함수이다.
    1. 만약 p 가 함수가 아니라면 fantasy-land/filter 의 동작은 명세되어있지 않다.
    1. p 반드시 true 또는 false를 반환한다. 만약 다른 값을 반환하면 fantasy-land/filter 의 동작은 명세되어있지 않다.

  1. fantasy-land/filter 는 반드시 같은 Filterable을 반환한다.

Setoid

setoid는 동등관계를 집합(또는 타입)이다. (정확히 무엇을 위해서 사용하는지 모르겠다.)

Definition

  1. 반사성, 반사관계가 유사할듯
  1. 대칭

fantasy-land/equals :: Setoid a => a ~> a -> Boolean

Setoid는 반드시 fantasy-land/equals 메서드를 가져야한다. 전달인자를 하나만 받는다.

a['fantasy-land/equals'](b)
  1. b 는 반드시 같은 Setoid여야 한다.(Setoid는 carrier타입이므로 실제 담은 값의 타입이 같아야한다는 뜻일듯하다.)
    1. if b 가 같은 Setoid가 아니라면 fantasy-land/equals 의 동작은 명세되어 있지않다.(false를 반환하는걸 추천한다.)
  1. fantasy-land/equals 반드시 true또는 false를 반환한다.

Ord

Ord 클래스는 모두 정렬된 데이터 타입을 위해 사용된다.(하스켈 패키지 참조)

반드시 Setoid 스펙이 구횬되어있어야 한다.

fantasy-land/lte :: Ord a => a ~> a -> Boolean

Ord 는 반드시 fantasy-land/lte 메서드를 제공해야한다. fantasy-land/lte 는 하나의 전달인자를 가진다. 

a['fantasy-land/lte'](b)

  1. b 는 반드시 같은 Ord 값이어야한다. (Carrier 타입이므로 담고있는 값의 타입이 같아야한다는 뜻일듯 하다)
    1. b 는 같은 Ord가 아니라면 fantasy-land/lte 의 동작은 명세되어있지않다. (false 반환을 추천)
  1. fantasy-land/lte 는 반드시 true 또는 false를 반환한다.
