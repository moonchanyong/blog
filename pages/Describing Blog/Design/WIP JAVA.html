public Alarm evaluate(Issue issue) {
    Alarm alarm = null;
    if (issue.getStatus() == Status.OPEN) {
        if (now.isAfter(issue.getDueDate())) {
            alarm = Alarm.DANGER;
        } else {
            alarm = Alarm.NOTICE;
        }
    } else {
         alarm = Alarm.INFO;

    }
    return alarm;
}

 public Alarm evaluate(Issue issue) {

    if (issue.getStatus() == Status.CLOSED) {
        return  Alarm.INFO;
    }

    if (now.isAfter(issue.getDueDate())) {
        return  Alarm.DANGER;
    }
    return Alarm.INFO;
}

DAO vs Repository (참조)

DAO의 인터페이스는 데이터베이스의 CRUD 쿼리와 1:1 매칭되는 세밀한 단위의 오퍼레이션을 제공한다. 반면 REPOSITORY는 메모리에 로드된 객체 컬렉션에 대한 집합 처리를 위한 인터페이스를 제공한다. DAO가 제공하는 오퍼레이션이 REPOSITORY 가 제공하는 오퍼레이션보다 더 세밀하며, 결과적으로 REPOSITORY에서 제공하는 하나의 오퍼레이션이 DAO의 여러 오퍼레이션에 매핑되는 것이 일반적이다. 따라서 하나의 REPOSITORY 내부에서 다수의 DAO를 호출하는 방식으로 REPOSITORY를 구현할 수 있다.(http://aeternum.egloos.com/1160846 )

JAVA Christmas - 20. Garbage collection

GC는 자바의 주요 특징이다.

개발자에게 메모리 관리에보다 앱의 기능적인 측면에 초점을 맞추도록 도와준다.

GC는 사용하지 않는 메모리를 반납하도록 하는 메카니즘이다. 이러한 메카니즘을 유지할 수 있도록 힙에서 안전하게 제거 될 수 있는지 결정을 한다.

자바 프로세스는 사용 할 수 있는 메모리에 따라 heap, metaspace(PermGen successor), JIT codecache, thread stacks, and shared 라이브러리에 메모리를 할당한다. 이 글은 metaspace와 codecache에도 GC 가 있지, Heap에 초점을 맞춘다.

Java heap은 아래와 같이 세영역으로 나뉜다.

Eden space는 new Object가 위치한다. GC에서 살아남은 Eden space의 객체들은Suvivor space로 이동한다. 만약 survivor space에서 오래 살아남아 threshold(직역-문지방, 의역-기준선)을 넘으면 tenured space로 이동한다.

객체들이 각 space들을 이동하는 과정은 generational 또는 ephemeral GC라고 알려져있다. 그리고 이 과정은 대부분의 객체의 라이프사이클이 빠르게 끝난다는 가정을 기반으로 만들어졌다. 이러한 이ㅠ가 minorGC와 majorGC라고 부르는 이유이다. minorGC는 Eden psace와 Survivor sapce를 합친 young space에서 동작하고, majorGC는 tenured space에서 동작한다.

Phase in a GC cycle

자바의 메모리 모델에 대해서 짧게 설명 한후 GC cycle에서 무슨일이 일어나는지 알아보자.

Marking

Marking phase는 어느 GC알고리즘에서든 처음에 일어난다. 이 과정에서 알고리즘은 힙에서 제거 가능한 객체들을 찾는다 .

마킹이 될 객체들을 선택하는 방법은 여러가지가 있다. 가장 간단한 방법은 잘 알려진 reference counter를 확인한다. 각 객체가 얼마나 많은 참조를 가지고 있는지 추적한다. 참조가 제거되면 reference counter 도 감소한다. 그리고 0이 되면 이 객체는 죽은 객체라고 마킹 된다. 이 방법은 순환참조에서 몇가지 제약이 있으며 어느 GC에서든 사용되지 않는다. GC알고리즘은 객체에 접근 할 수 있는지 확인하기 위해서 graph traversal 알고리즘을 사용한다.

GC 루트에서 참조되는 객체들을 추적하기 시작하고, 이후 모든참조를 찾을 수 있다. 여기서 찾을 수 있으면 모드 접근 가능한 객체라고 볼 수있다. 접근 불가능한 객체는 죽은 객체이며 gabage collected 될 수 있다고 마킹된
