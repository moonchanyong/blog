GraalVM Native Image(이하 그랄, 네이티브 이미지)

자바 코드를 standalone으로 실행 가능하게 해주는 AOT 컴파일러

이 실행파일은 application, 라이브러리, JDK를 포함하며 JVM에서 동작하지 않는다.

하지만 필요한 memory management와 다른 VM의 thread sheduling(Substrate VM이라 불리는)을 포

함한다. Substrate VM은 runtime컴포넌트로 deoptimizer, garbage collector, thread scheduling etc을 포함한다.

이를통해 프로그램은 startup time을 빨라지고, runtime메모리 오버헤드도 JVM에 비해 적어진다.

Substrate VM is a framework that allows ahead-of-time (AOT) compilation of Java applications under closed-world assumption into executable images or shared objects (ELF-64 or 64-bit Mach-O).

Native Image Generator: native-image

native-image는 application과 dependence, JDK를 포함한 classes를 처리하기위한 유티리티이다.

Tree shaking처럼 접근하는 코드만 GraalVM 컴파일러에서 AOT컴파일을 통해 native binary로 컴파일 한다.

native-image 는 JVM 언어에 대해 지원하고, 선택적으로 native image는 dynamic laguage로 실행이 가능하다.(JavaScript, Ruby, R, or Python) native-image 에 --language:<languageId> 를 AOT 컴파일도 가능하다.

Image Generation Options

native-image command line은 class path에 대한 정보가 필요하다.

-cp 이후 directory 리스트 또는 jar파일을 : 으로 분리하여 명시한다. main 메서드가 포함되는 class name은 마지막 arugument로한다.

또는, -jar 를 통해 manifest에 main 메서드를 명시 할 수 있다.

  • native-image [options] class 명령은 현재 directory의 class를 실행가능하도록 빌드하다.
  • native-image [options] -jar jarfile 은 jar파일을 빌드한다.
  • -cp and —class-path <class serach path of directories and zip/jar files> 다른 분산된 디렉토리 리스트 또는 JAR, ZIP을 찾을 수 있도록 해준다.
  • -D<name>=<value> 는 image generator를 위한 JVM의 시스템 프로퍼티를 세팅하기위해 사용한다.
  • -J<flags> 는 <flag> 를 image generator의 JVM에게 전달한다.
  • -O<level> 0는 optimization을 하지않고, 1은 기본 optimization을 한다 (1이 default)
  • -ea 와 -da 는 생성된 이미지에 대해서 assertions을 가능(enable) 또는 불가능(disable)하도록 한다.
  • —allow-incomplete-classpath

Limitations of AOT Compilation

자바 feature에서 허용되지않는 aot 컴파일이있다. 그리고 이것은 performance 이점을 얻을 수 없다.

네이티브의 실행을 위해 최적화를 많이 하려면, 정적 분석을 해야하는데, 이 때, closed world assumption(이 프로그램이 동작하는 범위)이 필요하다. 모든 classses또는 bytecode가 빌드타임에 런타임에 실행될 수 있다고 생각하는 코드의 범위다. (빌드타임에 추측 할 수 있는 런타임에 대해서만 최적화를 한다)
